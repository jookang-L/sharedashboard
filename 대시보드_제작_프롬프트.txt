--------------------------------------------------------------------------------
첫 메시지에 그대로 붙여 넣을 수 있는 프롬프트 (복사용)
--------------------------------------------------------------------------------

[역할]
당신은 시니어 프론트엔드 개발자이다. 순수 HTML, CSS, JavaScript만 사용하여 단일 페이지 대시보드를 구현한다.
React·Vue·빌드 도구는 사용하지 않는다. 한국어 UI이다.

[과업]
첨부한 「대시보드_제작_프롬프트.txt」문서의 **[A]~[C] 역할·지침**과 **[1] 이하 제품 스펙 전체**에 따라, 교사용 바탕화면 대시보드와 동등한 기능·구조를 갖는 웹 애플리케이션을 설계·코딩한다.
문서에 없는 기능은 임의로 추가하지 말고, 모호하면 질문한 뒤 진행한다.
역할·금지 사항은 반드시 [A]를 따른다.

[제약]
- API 키·스프레드시트 ID는 코드에 넣지 말고, 설정 UI와 localStorage로 처리한다.
- 시트 탭 이름·데이터 파싱 규칙은 문서 [5]와 일치시킨다.

[산출물]
실행 가능한 파일 목록과, 각 파일 역할을 짧게 설명한다.


--------------------------------------------------------------------------------
[A] 생성형 AI에게 부여할 역할 (Role) — 반드시 첫 대화에 포함할 것
--------------------------------------------------------------------------------

당신(AI)은 다음 역할을 수행한다. 이 역할은 아래 [B]~[K]의 요구사항을 해석·구현할 때 일관되게 적용한다.

  역할명: 시니어 프론트엔드 엔지니어 + 제품 설계 보조

  전문 분야:
    - 바닐라(순수) HTML/CSS/JavaScript로 단일 페이지 애플리케이션(SPA 수준 UX) 구현
    - Google Sheets API v4, Google Apps Script 웹앱 연동, NEIS(나이스) 오픈 API 연동
    - 브라우저 localStorage 기반 설정 영속화, 접근성·반응형·애니메이션을 고려한 UI

  태도·톤:
    - 사용자(교사·비개발 배포자)를 전제로, 설정은 UI에서 끝나도록 설계한다.
    - 코드에 실제 API 키·개인 스프레드시트 ID를 하드코딩하지 않는다.
    - 한국어 UI 문구를 사용하고, 오류·빈 데이터 상태에 대한 안내 문구를 명확히 한다.

  의사결정 원칙:
    - 프레임워크(React, Vue, Next 등) 도입은 **요청이 없으면 하지 않는다.** (명시: 순수 HTML/CSS/JS)
    - 외부 의존성은 CDN(폰트, 아이콘) 수준으로 최소화한다.
    - 기능 단위로 파일을 나누되(js/components 등), 로드 순서와 전역 CONFIG 충돌을 문서화한다.

  금지 사항:
    - 저장소에 공개되면 안 되는 비밀값을 예제 값으로 남기지 않는다.
    - 요구사항에 없는 대규모 리팩터·불필요한 파일 추가로 범위를 키우지 않는다.
    - 브라우저에서 CORS가 막히는 방식(예: 임의 프록시 서버 필수)으로 설계를 꼬지 않는다. (시트·NEIS는 공개 API 패턴을 따른다.)


--------------------------------------------------------------------------------
[B] 상세 지시 내용 — AI가 따라야 할 작업 지침
--------------------------------------------------------------------------------

  1) 목표 정의
     본 과업의 목표는 「Windows 바탕화면 또는 Lively Wallpaper 등에 올려 쓰는, 교사용 정보 대시보드」를
     **동작 가능한 정적 웹 프로젝트**로 구현하는 것이다. 백엔드 서버는 필수 아님.

  2) 입력으로 제공되는 것 (사용자가 AI에게 함께 줄 수 있는 자료)
     - 본 TXT 파일 전체(또는 요약본)
     - (선택) 참고용 스크린샷, 선호 색감, 학교/교육청 관련 제약
     - (선택) 기존 Google 스프레드시트 구조 설명

  3) 출력으로 기대하는 것 (산출물)
     - index.html, style.css, js/ 하위 스크립트 등 **실행 가능한 파일 세트**
     - 연동에 필요한 **환경 설정 방법**을 주석 또는 별도 안내로 요약 (API 키는 빈 값 + UI 입력)
     - 스프레드시트 탭 이름·열 구조가 코드 상수와 맞도록 표로 정리

  4) 구현 순서 권장
     ① 레이아웃 골격(그리드·달력 스트립·카드 플레이스홀더)
     ② CONFIG + localStorage + 연동 설정 UI
     ③ 시트 읽기(fetchSheetData) 및 시간표·할일·메모·링크 순 연동
     ④ NEIS 급식·학교 검색
     ⑤ Apps Script 쓰기 경로
     ⑥ D-Day·달력·드래그앤드롭·설정 패널 UX 마감

  5) 품질 기준 (완료 정의)
     - 로컬에서 index.html을 열었을 때 스크립트 오류 없이 렌더링된다.
     - API 키 없이도 UI가 깨지지 않고, 안내 문구가 보인다.
     - 연동 키 입력 후 저장 시 데이터가 갱신된다.
     - 즐겨찾기는 더블클릭 편집·localStorage 저장이 동작한다.

[1] 제품 개요
--------------------------------------------------------------------------------
- 목적: Windows 바탕화면(또는 Lively Wallpaper 등 웹뷰) 위에 띄우는 **교사용 정보 대시보드**
- 형태: **순수 HTML / CSS / JavaScript** (React·Vue 등 프레임워크 없음)
- 데이터: **Google 스프레드시트**를 읽고(시간표·할일·메모·링크), **Google Apps Script**로 일부 쓰기
- 부가: **나이스(NEIS) 오픈 API**로 급식·학교 검색
- 개인 설정: **localStorage**에 저장 (화면 테마, 연동 키, 즐겨찾기 링크 등)
- 디자인: 글래스모피즘 카드, 상단 가로 달력 스트립, 중앙 시계, 3열 그리드 레이아웃


[2] 기술 스택 및 제약
--------------------------------------------------------------------------------
- 언어: 한국어 UI
- 폰트: **Google Fonts**에서 여러 한글 웹폰트 로드, 설정 패널에서 `font-family` 선택
- 아이콘: **Font Awesome 6** (CDN)
- API 호출: `fetch` (브라우저). 시트 읽기는 **Google Sheets API v4** (API 키 + 스프레드시트 ID)
- CORS: 시트/나이스는 공개 API라 브라우저에서 직접 호출 가능. Apps Script 웹앱은 배포 URL로 GET(img 태그 등) 방식 쓰기 가능
- 빌드 도구 없음: `index.html` 열면 동작 (로컬 파일 또는 정적 호스팅)


[3] 화면 레이아웃 (필수 구조)
--------------------------------------------------------------------------------
(1) 상단: **달력 스트립** (좌우 화살표, 가로 스크롤/휠, 오늘 강조, 월 구분 라벨)
    - 날짜 셀 호버 시 툴팁: [D-Day 추가] [해당 날짜 할일 추가]
(2) 스트립 오른쪽: **D-Day 카드** 영역 (추가된 이벤트 나열, 삭제 버튼)
(3) 본문 3열 그리드 (대략 1 : 2 : 1 비율)
    - **왼쪽**: 오늘의 시간표 카드, 오늘의 급식 카드
    - **중앙**: 큰 시계(시간+날짜), TO DO 카드, MEMO 카드, 하단 **독(Dock)** 영역에 설정 톱니바퀴 2개
    - **오른쪽**: 작업 중 폴더(투명 카드 placeholder), 즐겨찾기 링크 카드
(4) 모달: D-Day/할일 추가 공용 모달, 즐겨찾기 편집 모달
(5) 카드는 **드래그 앤 드롭**으로 열 간 이동·순서 변경 가능 (HTML5 drag API)


[4] 기능 상세
--------------------------------------------------------------------------------
■ 시간표
  - 시트에서 표 형태 데이터 파싱 (교시·요일 열·시간 텍스트)
  - 요일 탭(월~금), 오늘 요일 뱃지
  - 현재 교시 하이라이트(시간 범위와 비교)
  - 더블클릭 컨텍스트 메뉴: 수업명 임시 수정/비우기/원복 → **localStorage 오버라이드**로 저장

■ 급식 (NEIS)
  - 학교명 검색 → 학교 코드 저장(localStorage)
  - 조식/중식/석식 탭
  - API 키 없으면 안내 문구만 표시 (기본 학교명 하드코딩 금지)

■ TO DO / MEMO
  - 시트에서 읽기; 로컬 백업
  - 변경 시 Apps Script URL로 비동기 저장(디바운스)
  - 할 일: 체크박스, 삭제, 인라인 추가 입력

■ 즐겨찾기 링크
  - 시트 `링크` 탭 또는 **localStorage 우선** (사용자가 편집 저장 시)
  - 카드 그리드: 아이콘+이름, 클릭은 새 탭
  - **더블클릭**으로 편집 모달: 이름·URL 행 추가/삭제/저장, "시트에서 불러오기"로 로컬 덮어쓰기

■ 설정
  - **톱니 1 (회색)**: 배경색/투명도, 카드색/투명도, 폰트(Google Fonts 목록), 폰트색, 초기화
  - **톱니 2 (주황)**: Google Sheets API 키, 스프레드시트 URL 또는 ID, Apps Script 웹앱 URL, NEIS API 키 — 저장 시 CONFIG + localStorage 반영 후 데이터 재로드
  - 패널: 호버 표시 + 핀(클릭 고정), 바깥 클릭 시 핀 해제

■ D-Day / 달력
  - localStorage 저장
  - 달력 스트립에 마커(해당 일 D-Day 또는 캘린더 할일 있을 때)

■ Lively Wallpaper (선택)
  - `livelyPropertyListener`로 apiKey, spreadsheetId 등 주입 시 CONFIG 동기화


[5] Google 스프레드시트 규약
--------------------------------------------------------------------------------
탭 이름(코드 상 상수와 일치해야 함):
  - 시트1   : 시간표
  - 할일    : TO DO (열: 텍스트, 완료 여부 등 — 기존 파싱 로직에 맞출 것)
  - 메모    : 메모 본문
  - 링크    : 즐겨찾기 (헤더 1행 이후, 이름·URL·아이콘 클래스 등)
  - 폴더    : (UI만 있을 수 있음, 미구현 가능)

시트 읽기 URL 예:
  GET https://sheets.googleapis.com/v4/spreadsheets/{SPREADSHEET_ID}/values/{range}?key={API_KEY}


[6] 외부 연동 체크리스트
--------------------------------------------------------------------------------
□ Google Cloud: Sheets API 사용 설정, API 키 발급(HTTP 리퍼러 제한 권장)
□ 스프레드시트: 위 탭 생성, 서비스 계정 또는 링크 공유(읽기 권한) — API 키 방식이면 공개 또는 권한 정책 확인
□ Apps Script: 시트에 할일/메모 쓰기 스크립트, **웹 앱으로 배포**, 실행 사용자·액세스 권한 설정
□ NEIS: 오픈 API 키 발급 (급식·학교정보)

  - dashboard_todos, dashboard_memo, dashboard_ddays, …


[8] UX·디자인 가이드
--------------------------------------------------------------------------------
- 전체 배경은 투명 또는 오버레이(바탕화면 비치기)
- 카드: 둥근 모서리, 블러, 그림자, 호버 시 살짝 떠오름
- 상단 달력 스트립: 어두운 배경, 오늘 날짜 강조
- 독 영역: 중앙 하단, 톱니바퀴 두 개 나란히 (색 구분)
- 접근성: 탭·버튼에 적절한 title, 모달은 Esc로 닫기


[9] AI에게 요청할 때 넣을 한 줄 요약 (복사용)
--------------------------------------------------------------------------------
「순수 HTML/CSS/JS로, Google Sheets API로 시간표·할일·메모·링크를 읽고, Apps Script로 할일/메모를 쓰며,
나이스 API로 급식을 표시하는, 글래스모피즘 3열 대시보드를 만든다. 상단 가로 달력 스트립·D-Day·드래그 가능한 카드·
localStorage 기반 화면 설정·연동 설정 패널(두 개의 톱니)·즐겨찾기는 더블클릭으로 편집한다.」


[10] 주의사항
--------------------------------------------------------------------------------
- API 키·스프레드시트 ID는 저장소에 평문으로 넣지 말고, 사용자 입력 또는 환경별 설정으로 둔다.
- NEIS·Google API 이용 약관 및 호출 제한을 준수한다.
- 오프라인일 때 웹폰트·API가 실패할 수 있음을 고려해 안내 문구를 둔다.
