■ 첫 메시지에 그대로 붙여 넣을 수 있는 프롬프트 (복사용)
--------------------------------------------------------------------------------

[역할]
당신은 시니어 프론트엔드 개발자이다. 순수 HTML, CSS, JavaScript만 사용하여 단일 페이지 대시보드를 구현한다.
React·Vue·번들러는 사용하지 않는다. 한국어 UI이다.

[과업]
첨부한 「대시보드_제작_프롬프트.txt」문서의 **[A]~[C] 역할·지침**과 **[1] 이하 제품 스펙 전체**에 따라,
「Jook Board」와 동등한 기능·구조를 갖는 웹 애플리케이션을 설계·코딩한다.
문서에 없는 기능은 임의로 추가하지 말고, 모호하면 질문한 뒤 진행한다.

[제약]
- Google API 키·스프레드시트 ID·NEIS 키·사용자용 Apps Script URL은 코드에 하드코딩하지 말고,
  연동 설정 UI + localStorage(dashboard_api_config)로 처리한다.
- 건의하기(피드백) 전송 목적지는 **배포자(관리자) 전용**이므로, 사용자 연동과 분리하여
  google-api.js에 FEEDBACK_WEBAPP_URL(및 필요 시 비밀키 상수)만 고정한다.
- 시트 탭 이름·데이터 파싱 규칙은 문서 [5]와 일치시킨다.

[산출물]
실행 가능한 파일 목록과, 각 파일 역할을 짧게 설명한다.


--------------------------------------------------------------------------------
[A] 생성형 AI에게 부여할 역할 (Role)
--------------------------------------------------------------------------------

  역할명: 시니어 프론트엔드 엔지니어 + 제품 설계 보조

  전문 분야:
    - 바닐라 HTML/CSS/JavaScript로 SPA 수준 UX
    - Google Sheets API v4 읽기, Google Apps Script 웹앱 쓰기(할일·메모·건의)
    - NEIS(나이스) 오픈 API(급식·학교 검색)
    - localStorage 기반 설정, 글래스모피즘 UI, HTML5 드래그앤드롭
    - (선택) Electron/Electron 호환 타이틀바·창 제어 브리지

  태도:
    - 비개발자도 설정 UI만으로 연동 가능하게 설계한다.
    - 저장소에 실제 API 키·개인 스프레드시트 ID를 예제로 넣지 않는다.
    - 한국어 UI, 빈 데이터·오류 시 안내 문구를 명확히 한다.

  의사결정:
    - 프레임워크 도입은 요청이 없으면 하지 않는다.
    - CDN은 웹폰트·아이콘 수준으로 최소화한다.
    - js/components 단위로 나누고, index.html에서 로드 순서를 문서화한다.

  금지:
    - 비밀값을 공개 저장소에 평문으로 남기지 않는다.
    - 요구 범위 밖의 대규모 리팩터로 산출물을 키우지 않는다.
    - CORS 때문에 반드시 자체 프록시가 필요한 설계는 피한다(시트·NEIS는 브라우저 직접 호출 패턴).


--------------------------------------------------------------------------------
[B] 상세 지시 — 작업 지침·완료 정의
--------------------------------------------------------------------------------

  1) 목표
     Windows 바탕화면·Lively Wallpaper·브라우저·(선택) Electron 창에 올려 쓰는
     교사용 정보 대시보드를 정적 웹으로 구현한다. 백엔드 서버는 필수 아님.

  2) 입력으로 줄 수 있는 자료
     - 본 TXT 전체
     - 참고 스크린샷, 색감, 학교/교육청 제약
     - 스프레드시트 구조 설명

  3) 출력
     - index.html, style.css, js/** 등 실행 가능한 세트
     - 연동 방법 요약(주석 또는 README 수준 — 단, 사용자가 요청하지 않으면 별도 MD 생성은 생략 가능)
     - 스프레드시트 탭·열 구조 표

  4) 구현 순서 권장
     ① 레이아웃(타이틀바·달력 스트립·3열 그리드·독)
     ② CONFIG + localStorage + 연동 설정 UI(api-settings.js)
     ③ 시트 읽기 → 시간표·할일·메모·링크
     ④ NEIS 급식
     ⑤ Apps Script 쓰기(할일·메모)
     ⑥ 카드 레이아웃·드래그·숨김·메모 뷰
     ⑦ 화면 설정(settings.js): 테마·폰트·폰트 크기·색
     ⑧ 건의하기(feedback.js) + 배포자 전용 URL
     ⑨ (선택) Electron 브리지·타이틀바 동작

  5) 완료 정의
     - 로컬에서 스크립트 오류 없이 렌더링
     - API 키 없어도 UI가 깨지지 않고 안내 표시
     - 키 저장 후 데이터 갱신
     - 즐겨찾기 더블클릭 편집·저장 동작
     - 폰트 크기 슬라이더 시 전역 글자 비율 변화(rem + --font-size-base)

[1] 제품 개요
================================================================================
- 이름(예시): Jook Board
- 목적: 교사용 바탕화면 정보 허브(시간표·급식·할일·메모·링크·D-Day)
- 형태: 순수 HTML/CSS/JavaScript, 단일 페이지
- 데이터: Google Sheets API로 읽기, Apps Script로 할일·메모 쓰기
- 건의하기: JSON POST로 배포자 지정 웹앱만 호출(사용자가 입력한 Apps Script URL과 무관)
- 부가: NEIS 급식·학교 검색
- 설정: localStorage(화면 테마·연동 키·즐겨찾기·학교 정보 등)
- 디자인: 글래스모피즘 카드, 상단 가로 달력 스트립, 중앙 시계, 3열 그리드, 하단 독


================================================================================
[2] 기술 스택 및 제약
================================================================================
- 한국어 UI
- 폰트: Google Fonts(CDN) + 설정에서 선택. 표시명은 한글(예: 노토 산스, 프리텐다드).
- 로컬 폰트: public/fonts/*.ttf, font-files.json 목록 + font-labels.json(한글 표시명),
  npm run build:fonts 로 js/local-fonts.js 생성
- 폰트 크기: 12~24px 슬라이더 → :root --font-size-base, html { font-size }, 하위는 rem 비율
- 아이콘: Font Awesome 6(CDN)
- 시트: Sheets API v4 (키 + 스프레드시트 ID)
- CORS: 공개 API 패턴 준수


================================================================================
[3] 화면 레이아웃 (필수)
================================================================================
(0) 상단 앱 타이틀바(브라우저/Electron 공통 UI 영역)
    - 브랜드명, (선택) 위젯 모드, 화면 크기 프리셋, 건의하기, 닫기
    - Electron일 때 electron-bridge.js로 창 크기·닫기·항상 위 등 연동

(1) 달력 스트립: 좌우 이동, 오늘 강조, 월 라벨, 날짜 셀 툴팁으로 D-Day/할일 추가 진입

(2) D-Day 카드 영역

(3) 본문 3열 그리드(약 1:2:1)
    - 왼쪽: 오늘의 시간표, 오늘의 급식
    - 중앙: 시계, TO DO, MEMO, 하단 독(설정 톱니 2개)
    - 오른쪽: 폴더/작업 영역 카드, 즐겨찾기 링크

(4) 카드 드래그앤드롭: 열 간 이동·순서 저장(cards-layout.js 등, localStorage 유지)

(5) 모달: D-Day, 할일, 즐겨찾기 편집, 건의하기, 설정 패널 등

(6) 설정 패널 두 종류
    - 회색 톱니: 화면 설정(settings.js) — 배경/카드 색·투명도, 폰트·폰트 크기·폰트 색, 강조 색,
      추천 테마 칩 + 사용자 테마 저장
    - 주황 톱니: 연동 설정(api-settings.js) — API 키, 스프레드시트, Apps Script URL(할일·메모), NEIS 키
      ※ 건의하기 URL은 여기 두지 않음(배포자 코드 고정)


================================================================================
[4] 기능 상세
================================================================================

■ 시간표 (timetable.js)
  - 시트 표 데이터 파싱, 요일 탭, 오늘 교시 하이라이트
  - 더블클릭 등으로 수업명 임시 수정 → localStorage 오버라이드

■ 급식 (meals.js + NEIS)
  - 학교 검색·코드 저장, 조식/중식/석식, 키 없으면 안내

■ TO DO / MEMO (todo.js, memo.view.js)
  - 시트 읽기, 변경 시 Apps Script로 디바운스 저장
  - 할일: 체크·삭제·추가

■ 즐겨찾기 (links.js)
  - 시트 또는 로컬 우선, 더블클릭 편집, 시트에서 불러오기

■ 화면 설정 (settings.js)
  - 테마 프리셋, 커스텀 테마 저장
  - 폰트 크기·색·글꼴·강조색
  - 폰트 목록 한글 라벨

■ 건의하기 (feedback.js)
  - POST JSON { secret, name, email, category, message } (필드명은 구현에 맞출 것)
  - URL = getFeedbackWebhookUrl() → FEEDBACK_WEBAPP_URL만 사용(사용자 APPS_SCRIPT_URL과 분리)
  - secret = 코드 상수, Apps Script WEBHOOK_SECRET과 일치

■ Lively Wallpaper (선택, main.js 등)
  - livelyPropertyListener로 키 주입 시 CONFIG 동기화

[10] 주의사항
================================================================================
- API 키·스프레드시트 ID는 저장소에 평문으로 넣지 말 것.
- 건의하기 비밀키·웹앱 URL은 배포 정책에 맞게 관리(공개 저장소에는 플레이스홀더만).
- NEIS·Google API 이용약관·호출 제한 준수.
- 오프라인·API 실패 시 사용자에게 안내.


================================================================================
[11] 배포자 전용 메모 (건의하기)
================================================================================
- 사용자가 연동 설정에 넣는 Apps Script URL은 **할일·메모용**이다.
- 건의하기는 반드시 **관리자 스프레드시트**로만 가야 하면,
  FEEDBACK_WEBAPP_URL(및 스크립트의 WEBHOOK_SECRET)을 배포 빌드에서만 설정한다.
- 브라우저에서 /exec URL을 GET으로 열면 doGet 미구현 시 오류 페이지가 날 수 있으나,
  POST doPost는 별개로 동작할 수 있다.
